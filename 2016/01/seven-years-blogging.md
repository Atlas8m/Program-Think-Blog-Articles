　　顺便插一句：昨天看到新闻说“维基百科迎接15周年生日”，才意识到俺博客的生日与维基百科同一天 :)<br/>

<h2>★2015年的变化</h2><br/>
<h3>◇关于“读者评论”和“读者来信”</h3><br/>
　　（关于这俩事儿，“5周年庆”和“6周年庆”的时候都提过了。今天再次唠叨一下）<br/>
　　在2013年或更早，俺基本上可以做到：处理每一条读者评论。但如今的评论数实在太多，已经无法做到逐一回复了。<br/>
　　至于邮件，就处理得更少了——因为评论可以被大伙儿看到，所以俺优先回复评论。<b>如果你有【重要且紧急】的邮件发给俺，建议你在发邮件之后，再到俺博客最新的一篇博文中，用【留言的方式】提醒一下。</b><br/>
　　在这里给大伙儿道个歉——俺的时间实在有限，列位看官多包涵。<br/>
<br/>
<h3>◇启用 BT Sync——用于分享“翻墙工具”、“电子书”、“博客离线浏览”</h3><br/>
　　说到2015年的变化，不得不提 BT Sync。<br/>
　　这玩意儿刚发布的时候，就有热心读者向俺推荐。俺观望了一年多之后，终于决定正式启用——目前用它来自动同步“翻墙工具、电子书、博客离线浏览”。<br/>
　　说到这里，再次赞扬一下 BTSync 这个分布式的网盘，因为它是基于 P2P 分布式的，可以实现如下好处：<br/>
<blockquote style="background-color:#DDD;"><b>无容量限制</b>——只要你的硬盘足够大，就可以分享足够多的东西；相比之下，大部分商业网盘都有容量限制。<br/>
<b>无流量限制</b>——参与下载的节点越多，速度越快；相比之下，大部分商业网盘都有流量限制。<br/>
<b>无政府审查</b>——即使美国政府也无法审查它上面的内容；咱们天朝政府也一样没辙）。<br/>
<b>无版权审查</b>——连政府都没辙，商业公司就更没辙；相比之下，很多商业网盘存在版权审查（甚至包括国外的 GDrive）。<br/>
<b>免翻墙</b>——至少到目前为止，BT Sync【无需】翻墙工具就可以联通。所以俺用它来分享翻墙工具，再合适不过了。</blockquote><br/>
<h3>◇博客全站启用 HTTPS 加密</h3><br/>
　　这项功能是俺盼了很多年的，Google 终于在2015年提供了。<br/>
　　有了此功能，大伙儿访问俺博客会更加安全——前提是“<b>记得以【HTTPS】的方式访问俺博客</b>”。<br/>
　　关于此功能带来的好处，具体可参见《<a href="../../2015/10/Blog-Enable-HTTPS.md">博客全程启用 HTTPS 加密</a>》一文。<br/>
<br/>
<h3>◇俺的 GitHub 帐号开始启用</h3><br/>

<h2>★2016年的展望</h2><br/>
<h3>◇“博文”方面</h3><br/>
　　2016年，首要任务之一是把之前很多没收尾的系列，尽量补上（俗称“填坑”）。<br/>
　　另外，俺想多写点 Linux 方面的扫盲教程。主要是因为：“隐私问题和政府监控”越来越严重了。比如说：Windows 的新版本越来越流氓了（不信的话，你可以搜一下关于 Win10 的隐私批评）。而操作系统是个人安全体系的基石，如果你用的操作系统不靠谱，其它的安全措施都免谈。<br/>
　　还有一个计划要做的是：在<a href="https://github.com/programthink" rel="nofollow" target="_blank">俺的 GitHub 帐号</a>上整理一个 FAQ。这事儿本来是2015年就应该做的，拖到了2016年，实在惭愧。<br/>
　　这个 FAQ 将汇总这几年常见的读者提问。既有技术方面的，也有非技术方面的。等到整理得差不多了，俺会专门发一篇博文通告大伙儿。<br/>
　　至于其它方面的博文，俺想到哪写到哪。（当然，也会适当参考读者提的需求和建议）<br/>
<br/>
<h3>◇“资源分享”方面</h3><br/>
　　俺会加大分享电子书的力度，会增加关于“科普”的大类，或许还有“文学”的大类（也可能拿现有的“文艺”大类改造一下）。<br/>
　　关于“科普”的电子书，俺还有几个细节没想好：比如是全部都放在一个“同步目录”下面，还是根据学科分 N 个“同步目录”；是只放科普类的电子书，还是也放比较专业的书籍（不知道专业书籍看的人多吗？）<br/>
　　“文学/文艺”这个大类也是比较难划分子目录的。比如“文学”的电子书，有很多维度（国别、作者、体裁、年代）。如何用多级子目录来组织才比较好，大伙儿可以集思广益（欢迎到博客上留言提建议）。<br/>
