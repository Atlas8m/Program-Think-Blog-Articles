<h2>★关于 GitHub 的界面宽度问题</h2><br/>
　　由于 GitHub 本身的界面布局比较狭窄，当你打开“电子书清单”的时候，某些表格会出现“横向滚动条”，看起来不爽。<br/>
　　一个不太彻底的解决办法是：<br/>
　　在“<a href="https://github.com/programthink/books" target="_blank">电子书清单</a>”的界面上，点击 README.wiki 这个链接，之后的界面会比原来略宽一些。<br/>
　　比较彻底的解决办法是：<br/>
　　可以考虑安装某些浏览器扩展（比如 <a href="https://en.wikipedia.org/wiki/Greasemonkey" rel="nofollow" target="_blank">GreaseMonkey</a> 或 <a href="https://en.wikipedia.org/wiki/Stylish" rel="nofollow" target="_blank">Stylish</a>），这类扩展可以帮你定制【某个具体网站】的 HTML 界面布局。<b>当然啦，前提是：你要略懂 CSS 样式表。</b><br/>
　　补充说明：<br/>
　　本文发出之后，已经有热心读者在评论中分享了一个现成的 Stylish 的配置（<a href="https://userstyles.org/styles/108591/github-wide" rel="nofollow" target="_blank">网址</a>）。有现成的，就无需自己写啦 :)<br/>
