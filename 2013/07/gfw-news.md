<h2>★连上 VPN Gate 服务器的经验</h2><br/>
　　前面都是准备工作，现在开始说重点——连接 VPN Gate 服务器的经验。<br/>
<br/>
<h3>◇要经常使用 VPN Gate</h3><br/>
　　VPN Gate 在运行时会定期刷新服务器列表，并保存在本地。这样就可以确保你本地的服务器列表始终是最新的。<br/>
　　俺建议每天至少运行一次。如果你有虚拟机的话，可以考虑把 VPN Gate 装到虚拟机中，就可以长期开着。即使装在虚拟机中，VPN Gate 也可以共享翻墙通道给实体机或其它电脑使用。具体请参见《<a href="../../2013/01/cross-host-use-gfw-tool.md">多台电脑如何共享翻墙通道</a>》<br/>
<br/>
<h3>◇无法获取服务器列表，如何解决？</h3><br/>
　　如果你经常运行 VPN Gate，通常不会出现此情况。万一你因为某些原因，连续好几天没有使用 VPN Gate，<b>有可能</b>会导致它再也无法获取服务器列表。<br/>
　　关于这种情况的处理，俺在4月份的教程《<a href="../../2013/04/gfw-vpngate.md">扫盲 VPN Gate——分布式的 VPN 服务器</a>》里，已经提到了解决办法。这里再啰嗦一下。<br/>
　　你需要先临时运行另一个<b>可用的</b>翻墙工具（比如：自由门、无界、世界通...）。运行好了之后，在 VPN Gate 界面的服务器列表左下方有一个"<b>代理设置</b>"的按钮，点击之后会出现一个设置代理的界面。<br/>
　　然后你就填写另外一个翻墙工具的代理。（无界默认端口是<code>9666</code>、自由门默认端口是<code>8580</code>、世界通默认端口是<code>8000</code>）填好之后，点确定。VPN Gate 就可以利用另一个翻墙工具联网。<br/>
　　这时候再点 "刷新列表" 按钮，服务器列表应该就会正常更新了。<br/>
　　<b>更新完之后，要记得把刚才修改过的代理设置恢复原状。</b><br/>
<br/>
　　如果你手头没有其它翻墙工具可用，也可以到 VPN Gate 官方提供的镜像站点，（免翻墙）重新下载一个。VPN Gate 官网每天都会更新安装包。最新的安装包里面，内置了最新的服务器列表。<br/>
<br/>
<h3>◇为啥有的服务器无法连接？</h3><br/>
　　很多同学碰到的情况是：服务器列表已经刷新了，但是无法连接到列表中的服务器。这可能有如下几种原因：<br/>
<br/>
　　<b>1. 该服务器已经关机 或 该服务器已经换 IP</b><br/>
　　因为 VPN Gate 的服务器，大都是由各个国家的志愿者维护的。有些服务器说不定就是外国网友的家用电脑。如果是家用电脑贡献出来当 VPN Server，就无法确保长期在线。即使能长期在线，IP 地址也不会是固定的。<br/>
　　如果你的 VPN Gate，服务器列表已经有好几个小时没有刷新，那么该列表上的服务器信息就不够新鲜（某些服务器可能已经下线或改过IP了）<br/>
<br/>
　　<b>2. 该服务器的 IP 已被封杀</b><br/>
　　第二种情况就是，该服务器已经被 GFW 列入“IP黑名单”。俺写的<a href="../../2009/05/how-to-break-through-gfw.md">翻墙入门教程</a>中，有介绍 GFW 常用的招数，其中之一就是“IP黑名单”。<br/>
<br/>
　　<b>3. 你的公网 IP 已被封杀</b><br/>
　　这种情况，大伙儿可能没有注意到。<br/>
　　比如说，俺连不上某个服务器，但是俺切换了自己的公网IP之后，<b>同一个服务器</b>又可以连上了。这种情况说明，VPN Server 本身没有进入黑名单，而是自己的公网IP进入了黑名单。<br/>
　　俺猜测，GFW 会对国内的公网IP进行某种统计。如果发现某个公网IP频繁连接某些国外的 VPN，并且这些连接具有某些特征，那么就会在一段时间内（具体多长，俺不清楚）封锁该IP对这些 VPN 的连接。在《<a href="../../2013/01/gfw-news.md">2013年1月翻墙快报</a>》中，俺有提到过，GFW 开始引入"基于行为特征的检测技术"。虽然 GFW 无法破解 VPN 的加密，但或许能根据特征来判断某个 VPN 连接是否用于翻墙。<br/>
<br/>
<h3>◇服务器无法连接，如何解决？</h3><br/>
　　刚才分析了连不上服务器的三种原因。下面分别说说应对方法。<br/>
<br/>
　　<b>1. 先判断服务器是否在线</b><br/>
　　很简单，你要经常刷新服务器列表，确保服务器列表经常处于最新状态。另外，要留意服务器列表中的“<b>登录数</b>”。登录数越大的，说明该服务器越活跃。<br/>
<br/>
　　<b>2. 判断“VPN 服务器”或“自己的公网 IP”是否被屏蔽</b><br/>
　　据俺观察，GFW 对 VPN Gate 连接的阻断，不是很及时。如果你眼神好，可以看出蛛丝马迹。<br/>
　　当你双击服务器列表上的某个服务器时，VPN Gate 首先显示“连接到某某服务器”，如果服务器能连上，会接着显示“正在验证证书”和“正在验证用户信息”（后面这两个一闪而过，眼神好能分辨出）<br/>
　　如果 VPN Gate 已经显示了“正在验证证书”或者“正在验证用户信息”，这时候突然断线，那么十有八九是被 GFW 阻断的。<br/>
<br/>
　　<b>3. 先切换服务器，不行再切换自己的公网IP</b><br/>
　　如果你仔细观察，感觉到有 GFW 阻断的迹象。那么你可以先尝试服务器列表上的其它服务器（尽量找登录数比较大的）。如果你连续尝试了很多服务器，都发现被 GFW 阻断，那么有可能是你的公网 IP 被屏蔽了。这时候就要考虑切换自己的公网 IP。<br/>
　　对于在家上网，切换公网 IP 比较容易。如果你用的是 ADSL 或 光猫 或 3G 上网卡，只要把上网设备断开（拔出或断电），稍等片刻，然后再连上。这时候你的公网 IP 应该就不同了。为了验证公网 IP 是否确实改了，可以用浏览器访问那些专门显示 IP 地址的网站。提醒一下：不同的宽带提供商，分配公网 IP 的方式不同。对于某些 ISP，这招可能不行（把上网设备重启动之后，得到的还是原来的 IP）<br/>
　　如果是在公司上网，你的公网 IP 也就是你公司的互联网出口 IP。这种情况下，你【难以】改变自己的公网 IP :(<br/>
